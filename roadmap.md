Карта оставшихся тем по Prisma ORM

1. Пагинация, сортировка и агрегаты (Построение фидов и аналитика)
   Пагинация: Разница между Offset-пагинацией (take + skip) и продвинутой Cursor-пагинацией (через cursor — как в бесконечных лентах соцсетей).
   Сортировка (orderBy): Как сортировать данные по датам, алфавиту или даже по полям из связанных таблиц.
   Агрегации и Группировка (\_count, groupBy): Как быстро узнать количество постов у юзера, посчитать средний возраст пользователей (\_avg) или сгруппировать данные для графиков в админке.

2. Транзакции ($transaction) — Безопасность данных
   Зачем это нужно: Бывают ситуации, когда нужно выполнить несколько действий разом. Если падает одно — должны отмениться все (атомарность).
   Sequential Transactions: Массив независимых запросов, выполняющихся друг за другом.
   Interactive Transactions: Самый гибкий вариант, когда внутри транзакции можно писать логику на JavaScript (if/else), читать данные и на основе этого принимать решения. Именно здесь мы научимся правильно создавать Пользователя одновременно с его Профилем.

3. Работа с миграциями и CLI (prisma migrate)
   Как правильно изменять схему в процессе жизни проекта, чтобы не потерять данные в продакшене.
   Разница между prisma migrate dev (для разработки) и prisma db push (для быстрых прототипов).
   Написание кастомных сид-скриптов (seed.ts) для автоматического наполнения базы тестовыми пользователями и постами при разворачивании проекта.

4. Сложные сценарии и Raw SQL (Когда возможностей ORM мало)
   Сырые SQL-запросы ($queryRaw / $executeRaw): Что делать, если нужно написать супер-сложный запрос с оконными функциями, который Prisma не умеет собирать сама.
   Prisma Extensions: Как писать расширения (плагины) для Призмы — например, автоматически хэшировать пароли перед сохранением в базу или делать "мягкое удаление" (Soft Delete), когда записи не удаляются физически, а просто помечаются флагом isDeleted: true.

5. Интеграция с Express.js и Обработка ошибок Prisma
   Архитектура: как правильно организовать слой сервисов/репозиториев в Node.js.
   Глобальный обработчик ошибок: Разбор кодов ошибок Prisma (например, P2002 — нарушение уникальности email, P2025 — запись для обновления не найдена). Мы напишем код, который будет перехватывать эти ошибки и красиво превращать их в понятные для фронтенда res.status(400).json({ error: "Этот email уже занят" }).
