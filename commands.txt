1) npx prisma init --datasource-provider postgresql --output ../generated/prisma 
Эта команда инициализирует Prisma в вашем проекте. 
Она создает базовый файл конфигурации schema.prisma и файл переменных окружения .env.

Флаг --datasource-provider postgresql заранее настраивает схему на работу с базой данных PostgreSQL.
Флаг --output ../generated/prisma переопределяет стандартную директорию 
(которая обычно находится в node_modules/.prisma/client) 
и указывает Prisma генерировать клиентский код по указанному пользовательскому пути.


2) npx prisma migrate dev --name init
если мы внесли изменения в таблицы, новая схема таблицы будет применения к базе данных
Создает и применяет миграции в процессе разработки, 
синхронизируя базу данных с вашей схемой (информация из нашей предыдущей беседы).


3) npx prisma generate
Эта команда генерирует или обновляет Prisma Client — код для выполнения запросов к базе
её необходимо запускать каждый раз, когда вы обновляете строку подключения в файле .env 
(например, при переходе на Prisma Postgres), чтобы новые настройки вступили в силу


4) prisma db pull 
используется для подключения к вашей текущей базе данных, чтения её структуры (таблиц, колонок, связей) 
и автоматического создания или обновления файла вашей схемы Prisma (schema.prisma).
Обратите внимание: хотя само название команды prisma db pull в текущих источниках напрямую 
не упоминается (это информация из моих внешних знаний), 
сам процесс там описан как интроспекция базы данных (introspection).
В источниках указано, что интроспекция — это один из обязательных шагов, 
если вы добавляете Prisma ORM в проект, где уже есть готовая база данных, 
и вам не нужно создавать таблицы с нуля. Рабочий процесс в таком сценарии (Сценарий B) 
выглядит следующим образом:
Установка Prisma CLI и других зависимостей в проект.
Интроспекция базы данных для генерации схемы Prisma (именно эту задачу и выполняет prisma db pull).
Генерация Prisma Client с помощью команды prisma generate 
(чтобы получить типизированный код для работы с базой).
Обновление запросов в коде приложения для использования Prisma ORM.
Если говорить проще, prisma db pull избавляет вас от необходимости вручную описывать каждую таблицу 
в файле schema.prisma. Она "вытягивает" (pull) готовое состояние вашей реальной базы данных прямо в 
конфигурацию Prisma.


5) npx prisma migrate dev --name schema_overhaul --force 
полный сброс и создание чистой миграции
