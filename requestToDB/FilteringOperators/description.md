//операторы фильтрации (Filter Operators). Они живут исключительно внутри блока where и используются для точечного поиска записей.
//Их можно разделить на три четкие группы в зависимости от того, с каким типом данных (строки, массивы или точечные значения) они работают.

//Строковые операторы (String Filters)
//Эти операторы работают строго с текстовыми полями (String) и помогают искать записи по частичным совпадениям

// 1) contains (Содержит)
//Что делает: Ищет вхождение подстроки внутри текста. Ему важно, чтобы указанные символы просто были внутри слова или фразы.
//2) startsWith (Начинается с)
//Что делает: Проверяет, что строка начинается строго с указанных символов.
//3) endsWith (Заканчивается на)
//Что делает: Проверяет, что строка заканчивается строго на указанные символы.

//Операторы сравнения и множеств (Equality & Set Filters)
//Эти операторы универсальны. Они работают с любыми плоскими типами (String, Int, Boolean, DateTime)
// и проверяют точное совпадение с одним значением или со списком значений.

//1) equals (Равно)
//Что делает: Проверяет строгое, стопроцентное совпадение.

//2) in (Внутри списка)
//Что делает: Принимает массив значений и ищет записи, у которых поле совпадает хотя бы с одним элементом из этого списка.

//3)notIn (Исключая из списка)
//Что делает: Полная противоположность in. Выплевывает из ответа все записи, которые совпали со значениями из массива.

//Операторы списков/массивов (Scalar List Filters)
//Эти операторы используются только тогда, когда само поле в твоей схеме базы данных является массивом

//1) has (Содержит элемент)
//Что делает: Проверяет, есть ли в массиве хотя бы один конкретный указанный элемент.

//2) hasEvery (Содержит абсолютно все)
//Что делает: Принимает массив и проверяет, что в поле базы данных присутствуют все до единого переданные элементы
